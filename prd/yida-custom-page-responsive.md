# 宜搭自定义页面增强：响应式 + 主题支持

> 创建时间：2026-03-12

## 需求概述

增强宜搭自定义页面的基础能力：默认支持响应式布局、内置多套主题风格（shadcn/ui、宜搭原生风格）、支持 history.push 路由跳转。

## 背景

当前宜搭自定义页面存在以下问题：
1. **响应式支持不足** - 页面在大屏/小屏表现不一致，移动端体验差
2. **缺乏主题系统** - 无法统一切换页面风格，每个页面需单独配置样式
3. **路由能力弱** - history.push 无法正常使用，多页面跳转体验差

### 2. 主题系统

#### 2.1 内置主题

| 主题名称 | 说明 | 风格 |
|----------|------|------|


### 3. 路由示例

```javascript
// 页面跳转示例
import { history } from '@yida/utils';

// 跳转新页面
history.push('/page2', { id: 123, name: '张三' });

// 监听路由变化
useEffect(() => {
  return history.listen((location) => {
    console.log('路由变化:', location.pathname, location.state);
  });
}, []);

// 获取参数
const { id, name } = history.location.state || {};
```

## 技术方案

### 技术选型

- **主题方案**：CSS Variables + React Context
- **响应式方案**：Tailwind CSS 类名 + 自定义断点 Hook
- **路由方案**：history 库 + 自定义 Router 组件

### 目录结构

```
/src
  /themes
    index.ts           # 主题入口
    default.ts         # 默认主题
    shadcn.ts          # shadcn 主题
    minimal.ts         # 极简主题
    enterprise.ts      # 企业主题
  /hooks
    useTheme.ts        # 主题 Hook
    useBreakpoint.ts   # 断点 Hook
    useHistory.ts      # 路由 Hook
  /components
    /Button            # 按钮（主题化）
    /Input             # 输入框（主题化）
    /Card              # 卡片（主题化）
    ...
```

### 主题切换原理

```tsx
// ThemeProvider 组件
function ThemeProvider({ children, theme = 'default' }) {
  useEffect(() => {
    document.documentElement.setAttribute('data-theme', theme);
    // 应用主题 CSS 变量
    Object.entries(themes[theme]).forEach(([key, value]) => {
      document.documentElement.style.setProperty(key, value);
    });
  }, [theme]);
  
  return <>{children}</>;
}
```
