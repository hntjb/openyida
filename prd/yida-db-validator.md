# 宜搭专属数据库规范检测工具

> 创建时间：2026-03-12

## 需求概述

开发宜搭专属数据库规范检测工具，自动扫描应用中的数据库配置、表单字段索引、数据关联等，给出优化建议和改进方案。

## 背景

宜搭专属版提供独立的数据库能力，但在使用过程中容易出现：
- **字段类型选择不当**：如用文本存储日期导致查询效率低
- **缺少索引**：常用查询字段无索引，查询慢
- **数据关联混乱**：表单间关联关系不清晰
- **命名不规范**：字段命名随意，后续维护困难

类似 MySQL 数据库规范检测，但适配宜搭场景。

## 功能清单

### 1. 数据库配置检测

| 检测项 | 说明 | 优先级 |
|--------|------|--------|
| 字段类型检测 | 检测字段类型是否合理 | P0 |
| 必填字段检测 | 检测必填字段配置 | P0 |
| 默认值检测 | 检测是否有合理默认值 | P1 |
| 唯一性检测 | 检测是否需要设置唯一约束 | P1 |

### 2. 索引规范检测

| 检测项 | 说明 | 优先级 |
|--------|------|--------|
| 索引缺失检测 | 检测常用查询字段是否建索引 | P0 |
| 索引冗余检测 | 检测是否存在冗余索引 | P1 |
| 复合索引检测 | 检测复合索引顺序是否合理 | P1 |
| 索引利用率 | 检测索引是否被使用 | P2 |

### 3. 数据关联检测

| 检测项 | 说明 | 优先级 |
|--------|------|--------|
| 关联表单检测 | 检测关联表单配置是否合理 | P0 |
| 外键完整性 | 检测关联数据完整性 | P1 |
| 级联操作检测 | 检测级联删除/更新配置 | P1 |

### 4. 性能规范检测

| 检测项 | 说明 | 优先级 |
|--------|------|--------|
| 大文本检测 | 检测是否滥用长文本字段 | P0 |
| 子表单深度 | 检测子表单嵌套层数 | P1 |
| 附件字段检测 | 检测附件字段是否过多 | P1 |
| 数据量预警 | 检测数据量接近上限的表单 | P1 |

### 5. 命名规范检测

| 检测项 | 说明 | 优先级 |
|--------|------|--------|
| 字段命名检测 | 检测字段命名是否规范 | P1 |
| 表单命名检测 | 检测表单命名是否清晰 | P1 |
| 重复字段检测 | 检测是否存在重复字段 | P1 |

## 检测规则示例

### 规则 1：日期字段类型

```json
{
  "rule": "date_field_type",
  "name": "日期字段类型检测",
  "description": "日期字段应使用日期类型，而非文本类型",
  "level": "warning",
  "detect": {
    "fieldType": "TextField",
    "fieldName": ["日期", "时间", "created", "updated"]
  },
  "suggestion": "建议将字段类型改为 DateField 或 CascadeDateField"
}
```

### 规则 2：缺少索引

```json
{
  "rule": "missing_index",
  "name": "常用查询字段索引检测",
  "description": "常用查询字段应建立索引以提高查询效率",
  "level": "warning",
  "detect": {
    "queryFrequency": "high",
    "noIndex": true
  },
  "suggestion": "建议在以下字段建立索引：状态、创建人、创建时间"
}
```

### 规则 3：大文本字段

```json
{
  "rule": "large_text_field",
  "name": "大文本字段检测",
  "description": "过长的文本字段会影响查询性能",
  "level": "error",
  "threshold": 2000,
  "detect": {
    "fieldType": "TextareaField",
    "maxLength": ">2000"
  },
  "suggestion": "建议拆分为多个字段或将内容存储到附件"
}
```

## 交付物

1. **检测工具 Web 应用** - 交互式规范检测界面
2. **检测规则库** - 内置 15+ 条检测规则
3. **优化建议生成器** - 自动生成优化方案
