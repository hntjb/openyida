# 宜搭应用多版本开发工具

> 创建时间：2026-03-12

## 需求概述

为宜搭应用提供多版本开发能力，支持版本管理、环境区分、灰度发布、回滚等企业级功能。

## 背景

企业使用宜搭时常遇到：
- **测试与生产混淆**：开发中的功能直接发布到生产环境
- **版本管理困难**：无法追踪每次变更历史
- **发布风险高**：无法灰度测试直接全量发布
- **回滚困难**：发布出问题只能手动修复

类似 Git 的版本控制能力，适配宜搭应用场景。

## 功能清单

### 1. 版本管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 版本创建 | 创建新版本（基于当前状态） | P0 |
| 版本列表 | 查看所有版本及变更记录 | P0 |
| 版本对比 | 对比两个版本的差异 | P1 |
| 版本回滚 | 回滚到指定版本 | P0 |
| 版本分支 | 支持分支开发（如 dev/test/prod） | P1 |

### 2. 环境管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 环境定义 | 定义开发/测试/预发布/生产环境 | P0 |
| 环境隔离 | 每个环境独立配置和数据 | P0 |
| 环境同步 | 从某环境同步配置到另一环境 | P1 |
| 环境变量 | 支持环境级别的变量配置 | P1 |

### 3. 发布管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 发布审批 | 发布前需审批确认 | P1 |
| 灰度发布 | 按比例/按用户灰度发布 | P0 |
| 定时发布 | 设置定时发布时间 | P1 |
| 回滚发布 | 一键回滚到上一版本 | P0 |
| 发布记录 | 完整的发布历史 | P0 |

### 4. 协作管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 多人协作 | 多人同时开发不同功能 | P1 |
| 变更审批 | 重要变更需要审批 | P1 |
| 权限控制 | 不同角色不同权限 | P1 |
| 操作日志 | 完整记录所有操作 | P0 |

## 技术方案

### 数据模型

```typescript
interface AppVersion {
  id: string;
  appType: string;
  version: string;        // '1.2.0'
  branch: string;         // 'main', 'dev', 'test'
  parentVersion: string;   // 父版本
  status: 'draft' | 'published' | 'archived';
  changes: Change[];
  createdBy: string;
  createdAt: Date;
}

interface Change {
  type: 'add' | 'update' | 'delete';
  target: 'form' | 'page' | 'flow' | 'permission';
  targetId: string;
  targetName: string;
  detail: any;
}

interface Release {
  id: string;
  versionId: string;
  targetEnv: 'test' | 'staging' | 'production';
  type: 'full' | 'gray';
  grayConfig?: {
    type: 'percent' | 'user';
    percent?: number;
    userGroups?: string[];
    schedule?: string[];
  };
  status: 'pending' | 'running' | 'completed' | 'failed' | 'rolled_back';
  createdBy: string;
  createdAt: Date;
  completedAt?: Date;
}
```

## 交付物

1. **版本管理 Web 应用** - 版本控制界面
2. **发布管理模块** - 灰度发布、回滚
3. **环境管理模块** - 多环境配置
4. **CLI 工具** - 命令行版本管理

## 备注

- 多版本功能主要面向企业版/专属版用户
- 灰度发布依赖宜搭的用户体系
- 版本数据存储在独立数据库
