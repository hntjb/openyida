# 宜搭 Bug 分析工具

> 创建时间：2026-03-12

## 需求概述

开发宜搭 Bug 分析工具，能够自动分析宜搭应用运行时的错误，快速定位问题原因并给出解决方案。如果无法自动解决，可一键跳转钉钉官方渠道提 VOC/工单。

## 背景

宜搭应用开发中常遇到各类问题：
- **错误信息模糊**：报错信息不明确，难以定位问题
- **排查效率低**：需要逐一排查配置、代码、数据
- **缺乏指导**：不知道如何解决，只能提交工单
- **工单填写繁琐**：提工单需要描述大量信息

类似 Sentry 的错误分析能力，但针对宜搭场景定制。

## 错误类型与解决方案

### 类型 1：JS 运行时错误

```
错误信息：Cannot read property 'xxx' of undefined
原因：尝试访问 undefined/null 的属性
解决方案：
1. 检查变量是否正确初始化
2. 使用可选链操作符：value?.prop
3. 添加空值判断
```

### 类型 2：API 调用错误

```
错误信息：formData.getFormData is not a function
原因：API 方法名错误或版本不兼容
解决方案：
1. 检查 API 名称是否正确
2. 查看官方 API 文档
3. 确认宜搭版本
```

### 类型 3：表单验证错误

```
错误信息：字段"xxx"校验失败
原因：字段值不满足校验规则
解决方案：
1. 检查字段的必填/格式校验配置
2. 查看表单数据是否满足规则
3. 调整校验规则或数据
```

### 类型 4：权限错误

```
错误信息：无权限操作该数据
原因：当前用户没有操作权限
解决方案：
1. 检查数据权限配置
2. 确认当前用户角色
3. 申请数据权限
```

### 类型 5：版本不兼容

```
错误信息：组件版本不兼容
原因：使用了低版本不支持的组件/属性
解决方案：
1. 查看宜搭版本更新日志
2. 升级宜搭版本
3. 使用兼容的替代方案
```

## 技术方案

### 错误收集 SDK

```javascript
// 在自定义页面中引入
import { YidaBugTracker } from '@openyida/bug-tracker';

YidaBugTracker.init({
  appType: 'APP_XXXXX',
  apiKey: 'your-api-key',
  collectError: true,
  collectApiError: true,
  onError: (error) => {
    // 上报错误到服务器
    reportError(error);
  }
});
```

### 错误分析引擎

```typescript
// 错误模式匹配
const errorPatterns = [
  {
    pattern: /(\w+)\.(\w+) is not a function/,
    type: 'api_error',
    analyze: (match) => ({
      cause: `方法 ${match[2]} 不存在`,
      solutions: [
        `检查方法名是否正确`,
        `查看官方 API 文档`
      ]
    })
  },
  // ... 更多模式
];
```

## 交付物

1. **Bug 分析工具 Web 应用** - 错误查看与分析界面
2. **错误收集 SDK** - 可嵌入自定义页面的错误收集器
3. **知识库** - 内置常见错误及解决方案

- VOC/工单跳转需要配置企业内部应用
- 错误信息会脱敏处理，保护用户隐私
- 复杂问题建议结合 AI 分析
